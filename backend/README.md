# backend

Go session manager for the Python `spcrawler` crawler.

## Responsibilities

- accepts crawl requests
- creates and tracks in-memory sessions
- launches the Python runner for each session
- keeps the latest full state snapshot
- streams live state updates over SSE
- stops or removes sessions without restarting the server

The backend is intentionally thin. Crawl logic lives in the Python package under `spcrawler/src`.

## Run

```powershell
cd backend
go run .\cmd\server
```

Default listen address:

```text
:8080
```

Override it with:

```powershell
$env:ADDR=":8090"
go run .\cmd\server
```

When `STATIC_DIR` is set, the server also serves the built frontend from that directory. That is how the single-container Docker and Heroku deployments work.

## API

- `GET /api/health`
- `POST /api/sessions`
- `GET /api/sessions`
- `GET /api/sessions/{id}`
- `GET /api/sessions/{id}/state`
- `GET /api/sessions/{id}/stream`
- `GET /api/sessions/{id}/events`
- `DELETE /api/sessions/{id}`
- `POST /api/sessions/{id}/remove`

Start request body:

```json
{
  "match": "CSK vs GT IPL 2026",
  "api_key": "your-gemini-key",
  "proxy_url": ""
}
```

`/stream` and `/events` both return the same SSE feed and emit `event: state`.

## Runtime Behavior

- On the first session start, the backend looks for `backend\scripts\.venv`.
- If the venv does not exist, it creates one with a system `python3` or `python`.
- It installs `backend\scripts\requirements.txt` into that venv.
- It launches `backend\scripts\run_scraper.py` with unbuffered stdout.
- The Python runner prints JSON snapshots, which the backend decodes and rebroadcasts.
- Session IDs are generated by the backend and included in the state stream.

## Requirements

- Go 1.22+
- Python 3.8+ available on `PATH`
- Network access for dependency installation and crawl traffic

If `crawl4ai` browser binaries are not installed yet, activate `backend\scripts\.venv` and run `crawl4ai-setup` once.

In Docker or Heroku, the image installs Python dependencies and browser binaries during the build step, so you do not need to run those manually.
